明白了，你的极简版 Ubuntu 根本没有 dhclient，也没有 netplan、NetworkManager，这意味着 无法用传统 DHCP 客户端自动获取 IP。所以我们只能靠 systemd-networkd 来配置网络。

下面我给你一个可直接使用的 systemd-networkd 配置示例，支持 NAT 或桥接模式的静态 IP 或 DHCP（如果有 DHCP 服务器）。

⸻

1️⃣ 创建 systemd-networkd 配置目录

sudo mkdir -p /etc/systemd/network


⸻

2️⃣ DHCP 自动获取 IP（前提：有 DHCP 服务）

创建文件 /etc/systemd/network/20-wired.network：

[Match]
Name=enp2s0

[Network]
DHCP=ipv4

保存后启用：

sudo systemctl enable systemd-networkd
sudo systemctl restart systemd-networkd
sudo systemctl restart systemd-resolved

检查 IP：

ip a
ip route
ping 8.8.8.8
ping www.baidu.com


⸻

3️⃣ 静态 IP 示例（适合 VMware NAT 或桥接模式）

创建文件 /etc/systemd/network/20-wired.network：

[Match]
Name=enp2s0

[Network]
Address=192.168.159.100/24   # 虚拟机 IP，和 VMware NAT 网关同网段
Gateway=192.168.159.2        # VMware NAT 网关，一般可查 VMware 网络设置
DNS=8.8.8.8

保存后启用：

sudo systemctl enable systemd-networkd
sudo systemctl restart systemd-networkd
sudo systemctl restart systemd-resolved

然后验证：

ip a
ip route
ping 8.8.8.8
ping www.baidu.com

⚠️ 注意：静态 IP 必须和 VMware 网络模式匹配，否则仍然会 Destination Host Unreachable。

⸻

💡 关键点总结：
	1.	极简 Ubuntu 没有 dhclient → 必须用 systemd-networkd
	2.	先确认 VMware 网络模式（NAT 推荐）
	3.	静态 IP 要和 VMware 网段一致
	4.	DNS 用 systemd-resolved 或直接写入 /etc/resolv.conf
